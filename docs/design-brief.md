# デザイン依頼ブリーフ: 障害対応訓練シミュレーター(仮称: やまびこ)

> **進捗メモ**(2026-07-12): Must 項目のゲーム画面・DOM 画面デザイン刷新は 6a「ナイトシフト・アーケード」として実装完了(`f5dca4a7`、トークンは `apps/web/src/styles/tokens.css`)。Should 項目の「ロゴ・ブランディング全面刷新」(ロゴ/ワードマーク SVG、favicon・アプリアイコン、ブランドガイド)は未着手。

## 1. 背景とビジョン

深夜0時、社内の誰も全容を知らない基幹サービス「やまびこ」が壊れる。先輩は寝ていて、上司は飲み会。プレイヤーは監視モニターとターミナルだけを頼りに、朝6時までにサービスを復旧させる——そんな障害対応訓練シミュレーターです。

今回の依頼は、**ゲーム本体(canvas製トリプルモニター画面)を含む全画面のデザインを、ゼロから統一的に構築すること**です。機能実装は進んでいますが、ビジュアルは仮のものです。canvas内のゲーム画面と、その前後を包むDOM画面(選択・ロビー・結果など)を別々にデザインするのではなく、**全体をひとつの「深夜のNOC(監視室)に置かれた社内システム」として、最初から地続きに設計してください**。canvasとDOMの境界をプレイヤーが意識しない状態がゴールです。

ロゴ・ブランディングも今回の刷新対象です。仮称「やまびこ」の名を冠する社内基幹サービスとしてのアイデンティティ(ロゴ、ワードマーク、ブランドカラー)を、画面デザインと同じ世界観のもとでゼロから設計してください。

## 2. ターゲット

- ソロモード: 障害対応の経験がない学生〜若手エンジニア。ゲーム感覚で始めて対応の「型」を学ぶ
- マルチプレイ: SREチーム・研修担当者。チーム訓練(GameDay)の代替として業務で使う。上司や講師が会議室のスクリーンに投影しても信頼感が保てることが必須

## 3. 世界観とトーン

- 舞台は深夜のNOC。時刻は深夜0時〜朝6時。蛍光灯は半分消えていて、モニターの光だけが強い
- 目指すトーン: 「本物の緊張感 + くすっと笑える余白」。ただしビジュアルはふざけない。夜勤の静けさと孤独感。ユーモアはコピーや状況(古いRunbook、寝ている先輩)が担うので、ビジュアルは真顔でいてほしい
- 声の人物像: 「冷静で頼れる夜勤の先輩。緊張する場面でも落ち着いていて、たまに乾いた冗談を言う」
- 全体の演出軸は「夜 → 朝」: フローが進むにつれて画面で夜が明ける。成功エンド「静かな朝」= 安堵と達成感、失敗エンド「にぎやかな朝」= インシデントレポートが発行されるが、恥や懲罰の演出は絶対にしない

### 避けたいもの(抽象語ではなく具体例で)

- ホラー表現: ジャンプスケア、血、ノイズだらけのグリッチ、不気味なキャラクター
- 「AIが作ったようなUI」: 紫グラデーション、グラスモーフィズム、均等に薄い配色
- 「汎用ダークダッシュボード」: ミント一色のアクセント、特徴のないカードの羅列
- 幼稚なゲーム感: コミック系フォント、バウンドするアニメーション、装飾過多なバッジ
- 可読性を犠牲にする演出: 強いCRT湾曲・走査線で文字が読みにくくなる状態

## 4. 技術的な制約(デザインの自由度は最大限確保する)

- ゲーム本体は canvas 描画で、論理解像度は 1920x1080 固定。**canvas内の見た目(モニターレイアウト、配色、タイポグラフィ、HUD、演出)はすべて今回のデザイン対象**。ただし描画は canvas API で実装するため、成果物はレイヤー構造とスプライト/描画仕様に落とせる形にすること
- ダークテーマ基調は維持。現行の基準色に縛られる必要はなく、トーンに合う基準色の再提案を歓迎
- アクセシビリティ: 本文フォント16px以上、タッチターゲット44px以上、コントラストAA以上、キーボードフォーカスの可視化。canvas内も同水準の可読性を確保する
- DOM側の実装は Preact + CSSクラス + CSS変数。スタイルは「デザイントークンの差し替え + クラス単位のCSS」で実装できる形に落とせること。canvas側も同じデザイントークンを参照する前提で設計する
- 録画されるのは canvas 内のみ。**canvas内のデザインはリプレイ・教材として単体で共有されても成立する必要がある**。DOM側の演出は録画物を汚さないので自由度が高い

## 6. 感情曲線(この起伏をデザインで支える)

好奇心(選択)→ そわそわ(ロビー)→ 緊張の立ち上がり(ブリーフィング)→ 緊張のピーク(プレイ)→ 安堵(復旧宣言)→ 静かな内省(ふりかえり)→ 達成感と朝の光(結果)→ 冷静な俯瞰(リプレイ)

canvas内はこの曲線の「緊張のピーク」を一手に担う。アラートの点灯、モニターの光量、ゲーム内時刻の進行など、canvas内の演出とDOM側の演出が同じ言語で緊張と弛緩を表現すること。

## 7. 参考にするもの・しないもの

- 監視カメラ系ゲームUI: 「一人で複数画面を見張る」構図と緊張感を参考に。ホラー要素は不採用
- 実在のNOC・Grafanaなどの監視ダッシュボード: 情報密度と機能美を参考に。無機質さは不採用
- 日本のオフィスの夜: 貼り紙、内線表、年季の入った備品などをディテールとして少量だけ。ダサさで可読性は落とさない

## 8. 成果物

- デザイントークン一式(色/タイポグラフィ/余白/角丸/影)— DOMとcanvasの両方が参照する共通トークンとして定義
- **ゲーム画面(canvas内)のデザインカンプ(1920x1080)**: トリプルモニターのレイアウト、各モニターの画面設計(ダッシュボード/ターミナル/コミュニケーション)、アラート・通知の状態別デザイン、夜→朝の環境光変化のキーフレーム
- 主要DOM画面のデザインカンプ(デスクトップ1440px基準、タブレット幅の挙動メモ付き)
- 共通コンポーネント: カード、ボタン、フォーム、タイムライン、状態バッジ
- 各画面の空/ローディング/エラー状態
- canvas実装者向けの描画仕様メモ(レイヤー構成、色・フォントのトークン対応表、アニメーションのタイミング)
- ブランドアイデンティティ一式: ロゴ・ワードマーク(SVG)、favicon/アプリアイコン、ブランドカラーとロゴ使用ガイド(最小サイズ、余白、ダーク背景での扱い)

## 9. 優先順位(MoSCoW)

- Must: デザイントークン、**ゲーム画面(canvas内)**、難易度選択、シナリオ一覧、結果
- Should: ロゴ・ブランディング全面刷新、リプレイ、ふりかえり、ロビー、共通トップバー、プレイ画面周辺パネル
- Could: ブリーフィングの演出強化
- Won't(今回はやらない): ライトテーマ、スマホ専用レイアウト

## 10. 成功基準

- スクリーンショット1枚(canvas内・DOM画面のどちらでも)で「深夜の障害対応もの」だと伝わる
- 企業研修でスクリーンに投影しても信頼感が保てる
- 選択画面 → canvasゲーム画面 → 結果画面と遷移しても、境界を意識させず世界観が途切れない
- canvas録画(リプレイ)を単体で見ても、教材として共有できる見た目の完成度がある
- 初見ユーザーが30秒以内に最初のシナリオを開始できる
